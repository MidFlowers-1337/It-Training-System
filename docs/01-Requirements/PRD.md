## 1. 背景与目标

### 1.1 项目背景

随着 IT 行业快速发展，企业和个人对技能培训的需求日益增长。传统培训机构面临以下痛点：

| 痛点 | 描述 |
|------|------|
| 选课盲目 | 学员不清楚学习路径，难以选择适合自己的课程 |
| 报名低效 | 手工登记、电话报名，名额管理混乱 |
| 信息不透明 | 课程信息分散，学员难以全面了解 |
| 缺乏个性化 | 千人一面的课程推荐，无法满足个性化需求 |
| 学习无追踪 | 学员学习进度无法追踪，缺乏激励机制 |

### 1.2 项目目标

构建一个基于 SpringBoot + Vue 的 IT 技能培训智能选课系统，实现：

| 目标 | 描述 | 衡量标准 |
|------|------|----------|
| **高效选课** | 学员可在线浏览、筛选、报名课程 | 报名流程 < 3 步完成 |
| **智能推荐** | AI 根据学习目标推荐课程和学习路径 | 推荐准确率主观满意度 > 70% |
| **名额精准** | 实时名额控制，杜绝超卖 | 超卖率 = 0 |
| **多角色协同** | 管理员、讲师、学员各司其职 | 三端功能完整可用 |
| **学习追踪** | 记录学习进度，提供学习报告 | 进度数据准确率 > 95% |

### 1.3 项目定位

- **类型**：毕业设计项目
- **规模**：中小型单体应用
- **用户量**：预计 < 1000 并发
- **亮点**：AI 智能推荐（Spring AI 集成）+ 学习管理系统

## 2. 用户角色与场景

### 2.1 角色定义

| 角色 | 英文标识 | 描述 | 典型用户 |
|------|----------|------|----------|
| 管理员 | ADMIN | 系统管理者，拥有全部权限 | 培训机构运营人员 |
| 讲师 | INSTRUCTOR | 授课教师，查看自己的班期和学员 | 培训讲师 |
| 学员 | STUDENT | 学习者，浏览课程、报名、获取推荐、学习 | IT 从业者/学习者 |

### 2.2 用户场景

#### 场景 1：学员选课报名

```
角色：学员小王（应届毕业生，想学后端开发）
目标：找到适合的 Java 课程并报名

流程：
1. 小王注册账号并登录系统
2. 浏览课程列表，按"后端开发"分类筛选
3. 查看"Java 基础"课程详情和班期
4. 选择周末班（还剩 5 个名额）
5. 点击报名，系统提示"报名成功"
6. 在"我的报名"中查看报名记录

期望结果：报名成功，名额减 1，收到确认信息
```

#### 场景 2：学员获取 AI 推荐

```
角色：学员小李（有 1 年工作经验，想进阶）
目标：获取个性化学习路径推荐

流程：
1. 小李登录后点击"智能推荐"
2. 输入学习目标："想成为全栈工程师，目前会 Java 基础"
3. AI 分析后返回推荐结果：
   - 推荐课程：Spring Boot 实战 → Vue3 前端 → 微服务架构
   - 推荐理由：基于现有 Java 基础，先巩固后端再拓展前端
   - 学习顺序：建议按推荐顺序依次学习
4. 小李可直接点击推荐课程查看详情并报名

期望结果：获得合理的个性化推荐
```

#### 场景 3：学员学习与打卡

```
角色：学员小张（已报名课程，正在学习）
目标：记录学习进度，获取成就

流程：
1. 小张登录后进入"学习中心"
2. 选择正在学习的课程，进入学习页面
3. 完成章节学习后，点击"完成本章"
4. 系统记录学习进度，更新进度条
5. 每日打卡，累计学习天数
6. 达成"连续学习7天"成就，获得徽章

期望结果：学习进度被记录，获得成就激励
```

#### 场景 4：管理员管理课程

```
角色：管理员张老师
目标：上架新课程并创建班期

流程：
1. 张老师登录管理后台
2. 进入"课程管理"，点击"新增课程"
3. 填写课程信息：名称、分类、难度、课时等
4. 保存后进入"班期管理"
5. 为新课程创建班期：选择讲师、设置时间、名额
6. 班期状态设为"报名中"
7. 查看统计看板确认数据更新

期望结果：课程和班期创建成功，学员端可见
```

#### 场景 5：讲师查看学员

```
角色：讲师王老师
目标：查看自己班期的学员名单

流程：
1. 王老师登录讲师端
2. 查看"我的班期"列表
3. 点击某班期查看学员名单
4. 导出或记录学员信息

期望结果：只能看到自己负责的班期和学员
```

## 6. 非功能需求

### 6.1 性能需求

| 指标 | 要求 | 测量方式 |
|------|------|----------|
| 页面加载 | 首屏 < 3s | Lighthouse |
| API 响应 | P95 < 500ms（非 AI） | 压测 |
| AI 推荐响应 | < 10s（含 AI 调用） | 日志统计 |
| 并发支持 | 100 并发用户 | JMeter 压测 |

### 6.2 安全需求

| 需求 | 措施 |
|------|------|
| 认证安全 | JWT + HTTPS |
| 密码安全 | BCrypt 加密存储 |
| 接口权限 | Spring Security 角色控制 |
| 防注入 | 参数校验 + MyBatis 预编译 |
| 防重复提交 | 前端按钮 loading + 后端幂等 |
| 敏感配置 | 环境变量管理 |

### 6.3 可用性需求

| 需求 | 要求 |
|------|------|
| 系统可用性 | 99%（演示环境） |
| 错误处理 | 统一异常返回，友好提示 |
| 降级策略 | AI 不可用时返回热门课程 |

### 6.4 兼容性需求

| 类型 | 要求 |
|------|------|
| 浏览器 | Chrome 90+、Firefox 88+、Edge 90+ |
| 分辨率 | 1920x1080 优先，1366x768 兼容 |
| 移动端 | 不要求（二期考虑响应式） |

## 8. 范围边界与约束

### 8.1 技术约束

| 约束 | 说明 |
|------|------|
| 后端框架 | Spring Boot 3.2.5 + Java 17 |
| 前端框架 | Vue 3.5 + Element Plus + PrimeVue |
| 数据库 | MySQL 8.0 |
| AI 服务 | Spring AI 1.0.0-M4（支持多模型切换） |
| 部署 | Docker Compose |

### 8.2 范围边界

#### 在范围内 (In Scope)

- ✅ 三角色权限系统
- ✅ 课程/班期/报名完整 CRUD
- ✅ 名额实时控制
- ✅ AI 智能推荐
- ✅ 混合推荐算法
- ✅ 学习进度追踪
- ✅ 学习计划管理
- ✅ 打卡与成就系统
- ✅ 简版统计看板
- ✅ 单体架构部署

#### 范围外 (Out of Scope)

- ❌ 支付功能
- ❌ 移动端 App
- ❌ 第三方登录
- ❌ 直播/视频播放
- ❌ 复杂审批流程
